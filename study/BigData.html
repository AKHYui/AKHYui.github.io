# 大数据处理架构Hadoop

# 概述

# Hadoop简介

  • Hadoop是Apache旗下的开源分布式计算平台
  • Hadoop核心是分布式文件系统HDFS和MapReduce

# Hadoop生态系统

# HDFS

  • Hadoop分布式文件系统，Hadoop核心之一

# HBase

  • 提供高可靠性、高性能、可伸缩、实时读写、分布式的列式数据库，采用HDFS作为底层数据存储

# MapReduce

  • 一种编程模型，用于大规模数据集的并行运算

# Hive

  • 一种基于Hadoop的数据仓库工具，对于hadoop的数据进行特殊查询、整理和分析存储

# Pig

  • 数据流语言和运行环境，适合Hadoop和MapReduce查询大型半结构化数据集

# Mahout

  • 帮助开发人员更方便的创建智能应用程序

# Zookeeper

  • 高效可靠的协同工作系统，提供分布式锁之类的基本服务，用于构建分布式应用

# Flume

  • 高可用、高可靠、分布式的海量日志采集、聚合和传输系统

# Sqoop

  • SQL-to-Hadoop的缩写，用于在Hadoop和关系数据库之间交换数据

# Ambari

  • 基于Web的工具，支持Hadoop集群的安装、部署、配置和管理

# 伪分布式的配置文件(必考)
  • core-site.xml
<configuration>
    <property>
        <name>hadoop.tmp.dir</name>
        <value>file:/usr/local/hadoop</value>
        <description>Abase for other temporary directories.</discription>
    </property>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:9000</value>
    </property>
</configuration>
  • hdfs-site.xml
<configuration>
    <proprety>
        <name>dfs.replication</name>
        <value>1</value>
    </proprety>
    <proprety>
        <name>dfs.namenode.name.dir</name>
        <value>file:/usr/local/hadoop/tmp/dfs/name</value>
    </proprety>
    <proprety>
        <name>dfs.datanode.data.dir</name>
        <value>file:/usr/local/hadoop/tmp/dfs/data</value>
    </proprety>
</configuration>

# 分布式数据库HBase

# HBase简介

  • HBase是面向列的数据库

# HBase数据类型

# 数据模型概述

  • HBase是一个稀疏、多维度、排序的映射表
# 索引：
  • 行键
  • 列族
  • 列限定符
  • 时间戳

# 数据库模型相关概念

  • 表(用表来组织数据，由行和列组成，可以划分为多个列族)
  • 行(HBase由若干行组成，每个行由行键标识)
  • 列族(一个表是多个列族的集合，是基本访问控制单元)
  • 列限定符(列族里的数据通过列限定符定位)
  • 单元格(通过行、列族、列限定符确定一个单元格)
  • 时间戳(每个单元格都保存着统一数据的多个版本，采用时间戳索引)

# HBase采用三级寻址

# 三层结构

  1. Zookeeper文件------记录了-ROOT-表的位置信息
  2. -ROOT-表------记录了.META.表中的Region信息
  3. .META.表------记录了用户数据表Region数据信息

# MapReduce

# MapReduce模型简介

  • MapReduce模型是以MS架构分而治之的

# Map和Reduce函数(必考)

# MapReduce模型的核心就是Map函数和Reduce函数

  • Map函数和Reduce函数都是以<key,value>作为输入的
函数 输入 输出 说明
Map <k1,v1> List(<k2,v2>) (1)将小数据集进一步解析成一批<key,value>对，输入Map函数中进行处理 (2)每一个输入的<k1,v1>会输出一批<k2,v2>,<k2,v2>是计算中间的结果
Reduce <k2,List(v2)> <k3,v3> 输入中间结果<k2,List(v2)>中的List(v2)表示同一批属于同一个k2的value

# MapReduce的各个执行阶段(必考)

# 执行过程

  1. 使用InputFormat模块做Map前的预处理，将输入文件切分为逻辑上的多个InputSplit
  2. InputSplit是逻辑切分而非物理切分，需要通过RecordReader(RR)处理InputSplit中的具体记录
  3. Map任务根据用户自定义映射规则，输出<key,value>作为中间结果
  4. 对Map输出进行分区、排序、合并、归并等操作，即洗牌(Shuffle)操作
  5. Reduce以<key,value-list>中间结果作为输入，输出给OutputFormat模块
  6. OutputFormat验证输出目录是否存在和输出结果类型是否符合配置类型

# HDFS系统结构是什么？主要职责是什么？

  • HDFS系统由多个节点构成，主节点和从节点
  • 主节点负责目录的创建删除重命名，并管理从节点和块的映射关系
  • 从节点负责根据名称节点进行创建删除重命名

# 块、元数据、命名空间概念

  • 块：HDFS主要以块为存储单位，Hadoop1.0一块为64M，Haddop2.0一块为128M
  • 元数据：FsImage用于维护系统树以及所有文件和文件夹的元数据，存储在名称节点中
  • 名称节点(NameNode)：负责管理分布式系统命名空间，包含FsImage和EditLog
  • 第二名称节点(Secondary NameNode)：负责进行FsImage和EditLog的合并操作和负责备份

# HDFS存取策略

  • 存储策略有数据存放、数据读取、数据据复制等方面
