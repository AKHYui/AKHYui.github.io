# 实用软件工程

# 软件工程概述

# 软件的概念及特点

# 概念

  • 软件不仅包括程序，还包括程序处理对象——数据，以及程序开发、维护和使用有关的图文资料，即文档。

# 特点

  • 软件是一种逻辑实体具有抽象性。
  • 软件和硬件不同，是人类智慧劳动的产物。
  • 软件在运行过程中不会磨损。
  • 软件至今未摆脱手工艺开发的方式。
  • 软件的开发和运行必须依赖特定的操作系统。

# 软件危机的表现与原因

# 软件危机

  • 指人们在开发软件与维护软件过程中遇到的一系列问题。

# 原因

  • 忽视开发前期需求分析。
  • 缺乏统一规范的指导。
  • 文档资料不全或不准。
  • 用户开发者缺乏交流。
  • 忽略测试重要性。
  • 不重视维护导致工作困难。
  • 专业人员缺乏经验。
  • 没有完善的质量保证体系。

# 软件工程目标和原则

# 目标

  • 达到要求的软件功能
  • 取得较好的软件性能
  • 开发出高质量的软件
  • 付出较低的开发成本
  • 需要较低的维护费用
  • 能按时完成工作，及时交付使用

# 原则

  1. 用分阶段的生命周期计划进行严格的管理
  2. 坚持进行阶段评审
  3. 实行严格的产品控制
  4. 采用现代化程序设计技术
  5. 软件工程结果应能被清楚的审查
  6. 开发小组人员应该少而精
  7. 承认不断改进工程实践的重要性

# 瀑布模型

  • 一种线性开发模型，具有不可回溯性。

# 优缺点

  • 优点：过程简单，执行容易。
  • 缺点：无法适应变更。

# 特点

  • 需求不发生或者很少变化，开发者一次性获得所有需求
  • 开发人员具有丰富的经验
  • 项目风险较低，不适合控制风险

# 快速原型模型

  • 优点：软件产品开发基本按线性顺序进行。
  • 缺点：所选用的开发技术和工具不一定符合主流的发展。

# 增量模型

  • 优点：将待开发软件系统模块化，一个开发周期内的错误不会影响到整个系统，开发顺序灵活。
  • 缺点：要求待开发软件系统可以被模块化。

# 螺旋模型

  • 优点：有助于把软件软件质量作为软件开发的一个重要目标。减少了过多测试或测试不足的风险。
  • 缺点：开发人员必须要有专业知识，否则会带来真正的风险。

# 喷泉模型

  • 特点：迭代和无缝。
