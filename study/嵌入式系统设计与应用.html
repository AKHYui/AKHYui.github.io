# Cortex-A8处理器的工作模式和状态(重点)

# 工作模式

处理器模式 模式标识符 备 注
用户模式 usr 正常程序执行模式
系统模式 sys 使用用户模式相同的寄存器组，用于运行特权级操作系统任务
管理模式 svc 系统复位或软件中断时进入该模式，常用于普通外部中断处理
外部中断模式 irq 低优先级中断发生时进入该模式，常用普通外部中断处理
快速终端模式 fiq 高优先级中断发生时进入该模式，常用于高速数据传输和通道处理
数据访问终止模式 abt 当存取异常时进入该模式，用于虚拟存储和存储保护
未定义指令终止模式 und 当执行未定义指令时进入该模式，用于支持硬件协处理器的软件仿真
安全监控模式 mon 可在安全模式和非安全模式下转换

重点：

  • 管理模式级别最高
  • 模式标识符
  • 除用户模式外，其他七种工作状态为非用户模式(特权模式)
  • 特权模式中，除系统模式以外的都是异常模式

# Cortex-A8处理器状态

# 有三种状态

  1. ARM状态：32位
  2. Thumb状态：16位或32位
  3. ThumbEE状态：16位或32位

用BX(带状态切换跳转指令)指令可以实现ARM状态和Thumb状态的切换。 PC最低位为0，切换ARM状态。PC最低位为1，切换Thumb状态

ARM指令必须在ARM状态下执行，Thumb状态同样如此，复位后处于ARM状态

# 寄存器组

Cortex-A8共有40个32位寄存器，包括33个通用寄存器和7个状态寄存器。状态寄存器组包括1个CPSR(当前程序状态寄存器)和6个SPSR(备份程序状态寄存器)。

# 通用寄存器(33个)

# 不分组寄存器8个

r0~r7

# 分组寄存器25个

r8~r122+r13~r146+r15

# 状态寄存器(7个)

CPSR 1个，SPSR 6个

6个异常模式对应6个SPSR，用于备份CPSR的值

# 异常处理

异常类型 优先级
复位 1(最高)
数据中止 2
FIQ 3
IRQ 4
预取中止 5
未定义指令SWI 6(最低)

# 异常响应(重点)

  • 保护断点：保存指令地址，送入堆栈。
  • 寻找中断入口：根据中断源，查找入口地址。
  • 执行中断处理程序
  • 中断返回：从中断处返回到主程序继续执行。

# 异常响应-硬件自动

  1. CPSR数据复制到SPSR中
  2. 设置CPSR状态位( (1). T位=0(ARM状态) (2). 设置异常模式 (3). 设置禁止位)
  3. 下一条指令存储到LR中
  4. 设置PC相应异常向量

# ARM指令寻址方式(重点)

# 立即寻址

ADD R0,R0,#1    ;R0<--R0 + 1
MOV R0,#0x00ff  ;R0<--0x00ff
  • 立即寻址效率高
  • MOV中的第一个必须是寄存器
  • #后必须是地址数
  • 立即寻址局限性较大

TIP

立即数 用二进制表示 顺序看，循环看 1和1的间隔大于8，则非法 小于8偶数个0是立即数 等于8两端0是否为偶数位，不是则非法

# 寄存器寻址

ADD R0,R1,R2    ;R0<--R1 + R2
MOV R0,R1       ;R0<-- R1
  • 效率比立即寻址低

# 寄存器间接寻址

LDR R0,[R1]     ;R0 <-- [R1]
  • 把存储器中的值给寄存器 加[]代表取存储器的值

# 寄存器移位寻址

MOV R0,R1,LSL #2        ;R0 <-- R1中的数左移两位
ADD R0,R1,R2,LSR #3     ;R0 <-- R1 + R2中的数右移三位
  • LSL(重点):逻辑左移，多出的丢掉，空位补0(例如：0x01逻辑左移3位 成为0x08 [0001 -> 1000])
  • LSR(重点):逻辑右移，多出的丢掉，空位补0
  • ASR：算术右移，针对有符号数，保证位数符号不变，正数高端空出位补0，负数补1
  • ROR:循环右移，从低端移出的位填入高端空出的为
  • RRX:带扩展的循环右移，操作数右移一位，高端空出用C位填充

# 变址寻址

LDR R0,[R1,#2]      ;R0 <-- [R1+2]
  • 将寄存器R1的值加上位移量2，然后传入R0中

# 数据处理指令

# MOV指令

MOV R0, #0x01       ;将立即数0x01装入R0
MOV R0, R1          ;将寄存器R1的值存入R0
MOVS R0, R1, LSL #3 ;将寄存器R1的值左移3位后传送到R0，并影响标志位
MOV PC, LR          ;将链接寄存器LR(R14)的值传送到PC中，用于子程序返回

# MVN指令

MVN R0, #Ox0FF      ;将立即数0xFF按位求反后装入R0，操作后R0=0xFFFFFF00
MVN R0, R1          ;将寄存器R1的值按位求反后传送到R0

# ADD指令

ADD R0, R0, #1      ;R0 = R0 + 1
ADD R0, R1, R2      ;R0 = R1 + R2
ADD R0, R1, R2, LSL #3  ;R0 = R1 + (R2 << 3)

# SUB指令

SUB R0, R0, #1      ;R0 = R0 - 1
SUB R0, R1, R2      ;R0 = R1 - R2
SUB R0, R1, R2, LSL #3  ;R0 = R1 - (R2 << 3)

# ADC指令

ADDS R0, R1, R2
ADC R1 ,R1, R3      ;用于64位加法，(R1,R0)=(R1,R0)+(R3,R2)
  • 将两个操作数相加后，再加上CPSR中的C标志位的值，将结果放入目的寄存器中

# AND指令(位保持)

AND R0, R1, R2      ;R0 = R1 & R2
AND R0, R1, #3      ;R0的位0和位1不变，其余清零

# ORR指令(位设置)

ORR R0, R0, #3      ;R0的位0和位1置1，其余不变

# EOR指令(位反转)

EOR R0, R0, #0F     ;R0的低4位取反

# BIC指令(位清除)

BIC R0, R0, #0F     ;将R1的低4位清0，其他不变

# CMP指令(比较指令)

CMP R1, R0          ;将R1的值减去R0的值，并根据结果设置CPSR标志位
CMP R1, #0x200      ;将R1的值减去0x200，并根据结果设置CPSR标志位
  • GT为>
  • LT为<

# TST指令

TST R1, #0x0F       ;检测R1的低4位是否为0
TST R0, #14         ;判断R0的4 3 2位是否为0 (14为1110)

# 什么是GPIO？
  • 通用可编程IO接口，有的芯片控制只要求一位(即开关)就够了，微控制芯片上都会提供这样的接口。
# GPIO端口有多少个？有多少组？分别为哪些？
  • 237个端口，分为15组，分别为GPA~GPJ和MP，可以引脚复用。
# GPIO主要寄存器有哪些？分别作用是什么？
  • 分为两类，正常模式和掉电模式。
  • GPxnCON--控制寄存器
  • GPxnDAT--数据寄存器
  • GPxnPUD--上拉/下拉寄存器
  • GPxnPUDPDN--掉电模式上拉/下拉寄存器

# 操作系统的体系结构

  • 单内核：集中式操作系统，整个系统为一个大模块，可以分为若干个逻辑模块，即处理器管理、存储器管理、设备管理和文件管理。
  • 微内核：把操作系统中的内存管理、设备管理、文件系统等尽可能的从内核分离出来，只保留少数的基本功能，利于维护和移植，但效率不高。

# 文件系统(重点)

  • 文件系统是操作系统用于确认磁盘上分区的文件的方法和数据结构。
  • 是操作系统的重要组成部分。

# 虚拟文件系统

  • 对多种实际文件系统共有功能的抽象，屏蔽了各种不同文件系统在细节上的差异。
  • 虚拟文件VFS确保对所有文件的访问方式都是完全相同的

# 常用Linux文件系统类型

  • ext2
  • ext3
  • ext4
  • yaffs
  • nfs
  • jffs2

# Root File System

  • 根文件系统是特殊的文件系统，是所有的文件系统结合成的一个完整的统一体。具备普通文件功能，也是内核启动挂载的第一个文件系统。
  • Linux启动时，第一个挂载的必须是根文件系统。
  • 一般说来，只有/bin，/dev，/etc，/lib，/proc，/var，/usr这些需要的，而其他都是可选的。
