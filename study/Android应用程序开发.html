# Android应用程序开发

# Android简介

# Android特征

  • Android系统提供了访问硬件的API库支持
  • 在内存和进程管理方面，Android具有自己运行时和虚拟机
  • Android提供丰富的界面控件给使用者
  • Android提供轻量级进程间通信机制——Intent
  • Android提供Service无用户界面、长时间后台运行的组件
  • Android支持高效快速的存储方式
  • Android支持位置服务地图应用

# Android生命周期

# 程序生命周期

# Activity生命周期

  • 一个界面对应一个Activity
  • onCreate()：Activity启动后的第一个被调用的函数，用来初始化Activity。
  • onResume()：与用户交互时被调用。
  • onDestroy()：程序主动调用finish()函数；程序被Android系统终结。

# LogCat

  • Log.v()：记录详细信息
  • Log.d()：记录调试信息
  • Log.i()：记录通告信息
  • Log.w()：记录警告信息
  • Log.e()：记录错误信息

# 界面布局

  • 界面布局使用xml声明

# 线性布局(LinearLayout)

  • 所有子元素都在垂直(vertical)或水平方向按照顺序在屏幕上排列。

# 框架布局(FrameLayout)

  • 用来存放一个元素的空白空间，只能放在左上角。

# 表格布局(TableLayout)

  • 将屏幕划分为表格，指定行和列将元素添加入表格。

# 相对布局(RelativeLayout)

  • 通过指定元素与其他元素相对位置关系，确定所有元素的布局位置。

# 绝对布局(AbsoluteLayout)

  • (不常用)通过指定坐标位置确定布局。

# 网格布局(GridLayout)

  • 将界面划分为网格，界面元素可以摆放在网格中。
