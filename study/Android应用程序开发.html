# Activity生命周期

  • 一个界面对应一个Activity
  • onCreate()：Activity启动后的第一个被调用的函数，用来初始化Activity。
  • onResume()：与用户交互时被调用。
  • onDestroy()：程序主动调用finish()函数；程序被Android系统终结。

# 界面布局

  • 界面布局使用xml声明

# 线性布局(LinearLayout)

  • 所有子元素都在垂直(vertical)或水平方向按照顺序在屏幕上排列。

# 框架布局(FrameLayout)

  • 用来存放一个元素的空白空间，只能放在左上角。

# 表格布局(TableLayout)

  • 将屏幕划分为表格，指定行和列将元素添加入表格。

# 相对布局(RelativeLayout)

  • 通过指定元素与其他元素相对位置关系，确定所有元素的布局位置。

# 绝对布局(AbsoluteLayout)

  • (不常用)通过指定坐标位置确定布局。

# 网格布局(GridLayout)

  • 将界面划分为网格，界面元素可以摆放在网格中。

# 组件通信与广播消息

# Intent

  • 一种轻量级消息传递机制
  • 可以讲一个组件的数据和动作传给另一个组件
  • 数据传递是双向的
  • 产生组件->接收组件->传递数据信息
  • 可以启动Activity和Service
  • 可以在系统发布广播消息

# 启动Activity

  • Intent可以实现不同Activity之间的切换和数据传递(分为显式启动和隐式启动)
  • 新的页面需要注册合法身份

# 显式启动

  • 为Intent指定当前应用上要使用的Activity
Button button = (Button) findViewById(R.id.btn);
button.setOnClickListener(new OnClickListener(){
    public void onClick(View view){
        Intent intent = new Intent(IntentDemo.this, ActivityToStart.class);
        startActivity(intent);
    }
});

# 隐式启动

  • 不需要指定需要启动哪个Activity，根据系统来决定
Intent intent = new Intent(Intent.ACTION_VIEW, Uri.parse("http://www.baidu.com"));
startActiity(intent);

# 后台服务

# Service(考点)

  • 完整的生命周期从onCreate()开始到onDestroy()结束
  • 通过调用Context.startService()启动Service
  • 通过调用Context.stopService()或Context.stopSelf()停止Service

# Service有两种使用方式

# 启动方式

  • 不能自启动，要通过其他组件启动，可以自行停止

# 绑定方式

  • Service的使用时通过服务链接实现的，绑定过程中Service没有启动，Context.bindService()会自启动Service，一个Service可以绑定多个服务链接

# Service有两种启动方式

# 显式启动

  • 用Intent指明Service所在的类，然后启动Intent
final Intent serviceIntent = new Intent(this, RandomService.class);
startService(serviceIntent);

# 隐式启动

  • 需要在注册Service时，声明intent-filter的action属性
<service android:name=".RandomService">
    <intent-filter>
        <action android:name="edu.hrbeu.RandomService" />
    </intent-filter>
</service>

final Intent serviceIntent = new Intent();
serviceIntent.setAction("edu.hrbeu.RandomService");

# 服务绑定

  • 绑定方式使用Service，要获取Service实例，正常启动Service并且调用其中的公有方法和属性，需要重载onBind()方法，并在其中返回Service实例
  • 当绑定意外断开的时候，系统将调用ServiceConnection中的onServiceDisconnected方法
  • unbindService()方法成功后并不会调用onServiceConnected()，onServiceConnected()仅在意外断开绑定时才调用

# 数据存储与访问

# 简单存储

# SharedPreferences

  • 一种轻量级数据保存方式
  • 开发人员可以将NVP(Name/Value Pair 键值对)保存在Android文件系统中

# 有三种访问模式

  • 私有(MODE_PRIVATE)
  • 全局读(MODE_WORLD_READABLE)
  • 全局写(MODE_WORLD_WRITEABLE)

TIP

定义访问模式

public static int MODE = MODE_PRIVATE;

# SP存储步骤(必考)

  1. 定义访问模式
  2. 定义文件名称
  3. 获取SP实例
  4. 调用put(type)(_, _);
  5. 调用commit(保存修改)
//设定访问模式为全局读和全局写
public static int MODE = Context.MODE_WORLD_READABLE + Context.MODE_WORLD_WRITEABLE;
//定义文件名称
public static final String PREFERENCE_NAME = "SaveSetting"
//获取SP实例
SharedPreferences sharedPreferences = getSharedPreferences (PREFERENCE_NAME, MODE);
//调用put(*type*)(_, _);
SharedPreferences.Editor editor = SharedPreferences.edit();
editor.putString("Name", "Tom");
editor.putInt("Age", 20);
editor.putFloat("Height", 1.81f);
//调用commit(保存修改)
editor.commit();

# SP读取(必考)

  1. 获取SP实例
  2. 调用get(type)(_, _);
SharedPreferences sharedPreferences = getSharedPreferences (PREFERENCE_NAME, MODE);
String name = sharedPreferences.getString("Name", "Default Name");
//Default Name是默认值，无法获取数值时使用
int age = sharedPreferences.getInt("Age", 20);
float height = sharedPreferences.getFloat("Height", 1.81f);

# 文件存储

# 内部存储

  • openFileOutput()函数为写入数据作准备而打开文件，获取输出流。
  • openFileInput()函数为读取数据做准备而打开文件，获取输入流。
  • 示例代码
String FILE_NAME = "fileDemo.txt";
FileOutputStream fos = openFileOutput(FILE_NAME, Context.MODE_PRIVATE);
String text = "Some data";
fos.write(text.getBytes()); //写入文件
fos.flush(); //将缓冲区写入文件
fos.close(); //关闭
  • 使用try/catch捕获可能产生的异常

# 数据库存储

  • DBOpenHelper继承了帮助类SQLiteOpenHelper，重载了onCreat()函数和onUpgrade()函数。
  • DBOpenHelper抽象差，不能实例化，需要新建一个类继承包。
  • 先建立和打开数据库。
  • SQLiteDatabase实例不能直接使用，需要声明。
  • 开发人员不应直接调用onCreat()和onUpgrade()，应由SQLiteOpenHelper类决定何时调用。
  • getWritableDatabase()函数用来建立和打开可读写数据库
  • getReadableDatabase()函数获得只读数据库
